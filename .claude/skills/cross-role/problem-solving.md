# Skill: /fis:problem-solving — Structured Problem Solving

> Kỹ thuật giải quyết vấn đề có hệ thống khi bế tắc. First principles thinking.

## Khi nào dùng
- Stuck trên problem phức tạp
- Need structured approach thay vì trial-and-error
- Debug issues mà không rõ root cause
- Break down overwhelming tasks

## Agent
Technical Analyst, Solutions Architect

## Prompt mẫu
```
/fis:problem-solving "App crashes randomly in production. No clear pattern. How to systematically find root cause?"
```
```
/fis:problem-solving "Need to process 10M records daily but current pipeline takes 8 hours. Budget limited."
```

## Nguyên tắc
- Define problem clearly trước khi solve
- Break down into smaller sub-problems
- First principles: question assumptions
- Multiple solution paths, then evaluate

## Verification
- [ ] Problem clearly defined
- [ ] Root cause identified (not symptoms)
- [ ] Solution addresses root cause
